Psychedelicatessen

Designing an inventory management system

Psychedelicatessen is a popular bagel shop for locals and students alike. The owner was still using Excel spreadsheets to track her inventory and was frustrated with manually tracking each of her ingredients. I outlined the requirements for a POS and inventory tracking system, that would address her concerns.

MY ROLE

System Analysis & Design


PROJECT DURATION

Fall 2017

THE PROBLEM

Psychedelicatessen is a breakfast and lunch eatery, serving bagels made from locally-sourced, organic ingredients. The owner, Laura Kerrone has been using an Excel spreadsheet to manage and track her costs but because it cannot integrate with her counter, it does not update in real time. The work must be done done manually, so managing the inventory and cost of each ingredient is difficult. Psychedelicatessen prides itself on locally-sourced ingredients, so it is important to Laura that the system be able to track the inventory by portion size and cost.

Her vision is a system that can track inventory usage and waste would help project future inventory needs, therefore reducing cost and waste, streamlining employee workflow, and leading to better service overall.

SYSTEM CAPABILITIES

The following capabilities were grouped into three main categories based off of Laura's requirements:


Inventory

Track inventory levels in different units
Store menu item, which are items which are sold, and the price * of the items
Build recipes for some menu items (ie. sandwich) from stored ingredients with ingredients in different units. Calculate the cost ** for each recipe
Track cost per ingredient
Convert units of measure between inventory units of measure (counting, buying) and recipe units of measure

Sales

Allow customers to customize recipes, and calculate the cost of the customized recipe
Print tickets for order assembly
Process payments
Allow employees to place and modify orders quickly, and with little training

Administrative

Keep track of waste and to help management implement recommend operational improvements
Track which employee placed orders, including employee logins
Generate reports
Maintain permission for users, for instance: Inventory management (admin), placing orders (employee & admin)

* Defined as the cost incurred by the seller/business
** Defined as the price payed by the buyer

BUSINESS BENEFITS

If the previous capabilities are met, Psychedelicatessen will experience a number of benefits:


  • Faster turnaround time at the register
  • Informed decision making based off the data
  • Fewer errors with reporting and inventory levels
  • Employees will no longer have to focus on manually tracking orders—instead they can focus on customer service
  • Accurate data collection will save money and time—no need for manual entry via Excel!

SYSTEM OVERVIEW

The system is built up of three subsystems: inventory, sales, and admin/reporting. However, I will focus on the two main subsystems, inventory and sales.

Inventory

This subsystem is where inventory items and recipes are maintained. Inventory items are accounted for, and cost and waste reports can be generated.

Sales

This subsystem is where the orders are created and fulfilled. Customers can customize their orders, menu items are maintained, and payments are accepted.

Admin/Reporting

This subsystem is dedicated to the admin, and consists of an admin dashboard and reporting tools. Those users with admin privileges will be able generate inventory and waste reports filtered by date.

USE CASES

The use cases are split up by subsystem and summarized in the table below. The full use cases for the inventory subsystem can be viewed in the next section. I focused in on the inventory subsystem because the problems were unique to this business, and many of the sales features were similar to a generic POS system.


Inventory Subsystem

Name Description
Maintain Inventory Item (includes cost, portion size conversion rules, and adding inventory items) The owner creates/reviews/updates/deletes inventory items, inventory costs, or inventory rules.
Maintain Recipe The owner creates/reviews/updates/deletes recipe.
Count Inventory Items The owner counts inventory and enters it into the system.
Generate Cost Report The inventory system generates the cost report and sends it to the owner.
Generate Waste Report The inventory system generates the waste levels with the cost and sends it to the owner.

Sales Subsystem

Name Description
Create Order The customer selects menu item and employee enters it into the system. System creates and records order, then prints the order ticket.
Fulfill Order The employee receives the order and makes the order. After the employee fulfills the order, the inventory level is reduced accordingly and updated.
Maintain Menu Item The employee creates a new menu item, edits an existing menu item, or deletes an existing menu item.
Accept Payment The employee takes payment and enters the amount into the system. Employee will also ask for customer’s email for promotional purposes.
Generate Sales Report The manager sets up the sales report to run on a schedule and selects the desired parameters. The system generates the sales report according to the requirements.

DOMAIN MODEL CLASS DIAGRAM

The most challenging part was determining how to update the ingredient levels after every order. Each ingredient bought by Psychedelicatessen was converted into portions with the corresponding cost per portion. Portions could look like 2 slices of bacon, 1 egg, or an 1/8 cup of spinach. These portion sizes would be used to create a recipe, and the recipes would become menu items. This way when a customer would purchase a menu item, the system could keep track of the inventory levels, and the cost per item. Additionally, the use of portions makes it easy to track customized orders because employees can add or remove portions with ease.

WIREFRAMES

REFLECTION


I was unable to get feedback from the owner, but I believe this solution fits her business needs. While I was designing this system, I began to question if an in-house solution was the best fit for Psychedelicatessen. There are already inventory management systems and all-in-one POS systems that exist, like Toast and Revel. Both systems can create recipes, manage inventory, track waste, generate reports, and handle sales. Pre-existing solutions are tried and tested, updated regularly, well supported, and implemented relatively quickly. However it relies heavily on the service, and that dependence could be crippling if the vendor is unreliable, or if they need to transition to another platform.

Because transaction data is so sensitive, I think it would best to use a trusted off-the-shelf solution. Without a dedicated IT team, it would be very difficult for Psychedelicatessen to create and maintain a secure, custom-made system. Many off-the-shelf solutions offer some level of customization to the business that could satisfy the needs of Psychedelicatessen.

It is also worth noting that some off-the-shelf solutions have APIs that can be used to analyze sales data. This could be a relatively quick and inexpensive way to gather more insights. It could be a great project for college students who live nearby to implement a dashboard with these insights. For example, a dynamic admin dashboard could be created that aggregates waste and inventory levels into one view or a weather API that could be integrated to determine if there is a correlation between the weather and type of sales. The possibilities are endless, and can be tailored to any future needs that arise.

My final recommendation would be to utilize a pre-existing solution with an API so that the owner can have the option of developing customized solutions with relatively low overhead.
